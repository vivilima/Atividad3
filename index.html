Controle de Versão

O controle de versão se tornou uma prática indispensável nos últimos anos, devido ao crescimento das equipes e a complexidade de desenvolvimento de softwares. Raramente temos um cenário de desenvolvimento que não seja colaborativo. Gerenciar conflitos de código, melhorar a comunicação e facilitar acompanhamento do que se está sendo feito é uma tarefa complexa e trabalhosa. Sendo assim, surgiram ferramentas de apoio ao desenvolvimento de software para dar suporte a estas atividades. Hoje uma das ferramentas mais usada para essa tarefa é o GIT. O próprio site do GIT o define como sendo um sistema de controle de versão, distribuído, open-source e desenvolvido para dar suporte de pequenos até grandes projetos com velocidade e eficiência. No site do sistema são mostrados projetos e companhias que o usam. Podemos citar como exemplo, gigantes do setor de TI como: Google, Facebook, Microsoft, Twitter entre outras. Para quem não está familiarizado com a ferramenta, no site do sistema GIT tem uma sessão para iniciantes que queiram aprender comandos, além de tutoriais de instalação.
